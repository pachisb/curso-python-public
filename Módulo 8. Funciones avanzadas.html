      # Funciones de primera clase
      En Python, las funciones son como cualquier otro objeto.
      Por tanto se soportan funciones de orden superior (funciones
      que reciben funciones como parámetros, o las devuelven).
      .
      Avanzado: Programación funcional (paradigma)
      : https://es.wikipedia.org/wiki/Programaci%C3%B3n_funcional
      .
      Otros: programación imperativa (la más común), funcional, lógica...
      Un lenguaje puede implementar uno o varios de esos paradigmas

      Al ser objetos, las funciones pueden (y suelen) tener atributos!
      .
        >>> abs.__doc__
      Nos devuelve la Docstring (o None si no está definida)
        >>> abs.__dir__()
      Devuelve lo equivalente a dir(f)
        >>> type(f)
        function

      # Funciones Lambda (anónimas)
      Avanzado (...)
      : https://en.wikipedia.org/wiki/Closure_(computer_programming)#Anonymous_functions
      .
      Ejemplo:
        lambda x: x + 1
        (lambda x: x + 1)(2)
        f = lambda x: x + 1  # podemos darle un nombre, para entenderlo mejor
        f(2)  # es lo mismo, y equivalente a si la hubiéramos definido "normal" con def
      .
      En Python, en resumen, son funciones sin nombre escritas en una notación breve,
      que siempre devuelven un valor (funciones "tipo matemático" o "productivas")
      .
      Largo pero bien explicado, interesante y con ejemplos:
      : https://realpython.com/python-lambda/

      # Clausuras (closures)
      Muy avanzado (...)
      
      Ejemplo:
        def by_factor(factor):
            def multiply(number):
                return factor * number
            return multiply  # será una función con 1 parámetro ("number")!
        f = by_factor(2)
        f(5)
      .
      En resumen y simplificado (...), funciones que devuelven otras funciones
      (definidas dentro de ellas) a las que se les ha "fijado" algún parámetro
      Es decir, tienen variables "capturadas" (función con un "estado" / valores)
      : https://realpython.com/python-return-statement/#returning-functions-closures
      Son un caso particular de funciones de orden superior (devuelven funciones)

      
      # Ejercicio / demo: lambda y composición
      
Función que reciba a su vez dos funciones (como parámetros) y
 /NOBR/
      nos devuelva otra cuyo efecto es aplicar anidadas las otras dos.

      NOTA: puede ser más sencillo de entender empezando por una que
      simplemente haga el efecto (aplicar ambas) pero no devuelva
      otra función. Solución real completa en la siguiente página
      Ejemplo:
        def f1(l): return l[0]
        def f2(s): return s.upper()
        def f_comp(primero, segundo, x):
            intermedio = primero(x)
            final = segundo(intermedio)
            return final  # o directamente: segundo(primero(x))
      .
        f_comp(f1, f2, l)

      Solución:
        def f_comp_creator(primero, segundo):
            # devolvemos una función!
            f = lambda x: segundo(primero(x))
            return f
      .
        f_comp_creator(f1, f2)(l)
        # lo mismo pero dándole un mombre y en dos pasos:
        new_func = f_comp_creator(f1, f2)
        new_func(l)

      ## zip()
        zip(*iterables)

      .
      ## map()
        map(function, iterable, ...)
      "Return an iterator that applies function to every item of iterable (...)"
      Más detalles en:
      : https://realpython.com/python-map-function/

      Ejemplo:
        def square(number):
            return number*number
        numbers = [1,2,3,4,5]
      .
        # Sin map(), forma "sencilla", sería:
        squared_numbers = []
        for number in numbers:
            squared = square(number)
            squared_numbers.append(squared)
      .
        # Lo mismo del bucle pero con map() sería:
        squared_numbers = map(square, numbers)

      ## reduce()
        reduce(function, sequence[, initial])
      "Apply a function of two arguments cumulatively to the items
      of a sequence (...)"
      .
      Es decir, los va "procesando" por pares progresivamente, con la
      función f que le indiquemos como argumento: f(f(f(x), y), z)...
      OJO: por tanto, no devuelve un iterador sino ya el result. "final"
      Ejemplo:
        # no es built-in (import!), pero está en la bibl. estándar
        from functools import reduce
        l = [3, 6, 8, 9, 2, 4, 5]
        product = reduce(lambda a, b : a*b, l)

      ## filter()
        filter(function, iterable)
      "Construct an iterator from those elements of iterable for which function returns True"
      .
      Es decir, nos sirve para filtrar (eliminar) los que no cumplen alguna condición
      Ejemplos:
        # filter devuelve un iterador, por eso al final lo convierto a list
        iter = filter(str.islower, ["hola", "Hello", "abc", "ABC"])
        new_list = list(iter)

      Para todas las funciones anteriores, salvo zip(), es habitual usar funciones
      lambda para pasar como argumento una función anónima ("temporal" o
      auxiliar, que se define justo donde la usamos por claridad y brevedad).
      Pero no tenemos por qué... podemos definir las nuestras.
      Ejemplo:
        from functools import reduce
        l = [3, 6, 8, 9, 2, 4, 5]
        # product = reduce(lambda a, b : a*b, l)
      .
        # lo equivalente sin lambda sería
        def mult2(a, b):
            return a * b
        product = reduce(mult2, l)
      .
      Más detalles en un artículo sencillo con ejemplos: aquí

      A veces es más sencillo usar funciones ya existentes (no lambda), como alguna del
      módulo operator, que nos da "equivalentes" para cosas como los operadores, que
      son algo parecido pero no funciones (no tienen nombre). Por ejemplo +, ==, <,  etc.
      Ejemplos:
        import operator
        from functools import reduce
      .
        factorial = reduce(operator.mul, range(2, 11))  # factorial de 10
        new_list = reduce(operator.concat, ["hola", "Hola", "abc", "ABC"])
        suma = reduce(operator.add, [3, 6, 8, 9, 2, 4, 5])
      .
        people = [ {'nombre': 'Peter', 'apellido': 'Parker'},
                   {'nombre': 'John', 'apellido': 'Smith'} ]
        sorted(people, key=operator.itemgetter('apellido'))

      ## list.copy(), dict.copy(), etc.
      Devuelven una copia superficial ("shallow", de primer nivel sólo) de
      la estructura original.
      Esto puede ser un problema (modificar lo copiado, afecta a lo original!)
      Si queremos copia "completa" o profunda ("deep"; recursiva!), usar
      el módulo copy de la biblioteca estándar. Ejemplo:
        import copy
        x = copy.copy(y)        # make a shallow copy of y
        x = copy.deepcopy(y)    # make a deep copy of y
      .
      Más detalles en la doc. oficial y también (más sencillo, recom.) en:
      : https://realpython.com/copying-python-objects/

      ## reversed()
        Init signature: reversed(sequence, /)
        Docstring:      Return a reverse iterator over the values
                        of the given sequence.
      .
      Ejemplo:
        reversed([1, 2, 3])
        list(reversed([1, 2, 3]))

      ## sorted()
        sorted(iterable, /, *, key=None, reverse=False)
      Devuelve otra estructura (nueva) con los elementos de la primera ordenados
      NOTA: los argumentos 'key' y 'reverse' son opcionales y por nombre siempre
      .
      Usamos key (una función que devuelve algo "ordenable") cuando queremos un orden
      "a medida" (ej. con estr. de datos que no lo tienen o podemos querer otro: dict - keys).
      Ejemplo con un caso sencillo para entenderlo (listas):
        l = ['j', 'b', '0', 'W', 'efg', 'zz', 'z', 'za']
        sorted(l)  # ¿qué ocurre? ¿por qué?
        sorted(l, key=str.casefold)  # o lower
        sorted(l, key=len)

      Ejemplos más complejos con diccionarios (por def. se ordenan por clave):
        diccio = {'0' : 'W', 'b' : 'efg', 'j' : 'z', 'h' : 'z', 'za' : 'zz'}
        sorted(diccio)
      .
        def comp_dicc_value(diccitem):
            # se recibe un item (tupla c-v), y se devuelve el valor
            return diccitem[1]
        sorted(diccio, key=comp_dicc_value))  # ordenados por valor
      .
        sorted(diccio.items(), key = lambda x: x[1])  # lo mismo con lambdas
      .
      Y si puede haber valores repetidos, podemos "desempatar" con las claves:
        sorted(diccio.items(), key = lambda x: x[1] + x[0])  # v+c concatenados

      Avanzado: otros usos de ordenación con parámetro key:
      .
      Clases, para ordenar por alguno(s) de sus atributos, usando getattr()
      de forma similar a como habíamos usado operator.itemgetter() (...)
      : https://realpython.com/python-sort/#sorted-with-a-key-argument
      .
      También para ordenar por ejemplo cadenas que contienen números.
      Tener en cuenta que el orden alfabético para cadenas no es igual
      que para números. Ejemplo:
        1 < 2 < 9 < 10 < 11
        "1" < "10" < "11" < "2" < "9"  # se mira carácter a carácter!

      ## list.sort()
      Equivalente a sorted() que ordena una lista (la modifica!), y es más eficiente.
      OJO, devuelve None para indicar que altera la lista "in place", no crea otra.
      .
      Opcional: argumento key (por nombre siempre). Ejemplo:
        l.sort(key=list.upper)
      Igual que sorted(), también soporta el argumento reverse opcional (por nombre)
      .
      Algoritmos de ordenación en Python:
      : https://docs.python.org/3/howto/sorting.html#sortinghowto
      IMPORTANTE: es ordenación estable (no altera orden para los iguales)
      Eso significa que podemos aplicarla varias veces (orden por varios criterios!)

      Curiosidades / bromas
      Ineffective sorts:
      : https://xkcd.com/1185/
      : https://gkoberger.github.io/stacksort/

      # Generadores y yield
      Avanzado: saber al menos usarlos, no definirlos!
      "A generator is a recipe for producing values"
      Ejemplo:
        def gen():
            for n in range(12):
                yield n ** 2
        list(gen())
      .
      yield es similar a return pero el estado se queda "guardado" para la sig. invocación
      Se comportan como iterables (produciendo los elementos a demanda / tipo "streaming")
      .
      Más detalles y ejemplos, muy claro y recomendable:
      : https://realpython.com/introduction-to-python-generators/

      Si la función acaba "normal" (return explícito o se llega al final),
      se indica que ya se ha "agotado" el generador.
      Los bucles, etc. esperan/comprueban justo esto!
        def gen():
            for n in range(25):
                if n == 10:
                    return  # caso especial, volvemos "del todo"
                yield n ** 2
        
        for i in gen():
            print(i)
        # y en cualquier sitio donde valga un iterable, también:
        print(list(gen()))

      @CLASS@ ejercicio-2

      Función "estándar" que calcula la n-ésima potencia de un número,
      punto de partida para adaptarla y obtener un generador:
        def potencia(base, exponente):
            resul = 1
            i = 1
            while i <= exponente:
                resul = resul * base
                i = i + 1
            return resul
