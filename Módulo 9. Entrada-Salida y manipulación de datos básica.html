      # Ficheros de texto y binarios
      Ficheros de texto: separados por saltos de línea
      Caso partic.: ficheros "imprimibles": sin algunos caract. especiales
      Sencillos, legibles, sin "estructura" ni "formato", más
      robustos... Ojo con los encodings! Editor de texto (...)
      .
      Ficheros binarios: los que NO son de texto
      .
      ## Formatos de salto de línea
      LF (Linux, Web...) recomendado
      CR (Apple, en desuso)
      CRLF (Windows, 0x0d, 0x0a, en ese orden!)
      No mezclar nunca, puede dar muchos problemas!

      # Formatos de fichero básicos
      ## txt
      Lo que "nosotros queramos" que contenga (siempre que no sea un fich. binario)
      Convención: usar esta extensión para texto legible (para personas...)
      Otros ficheros de tipo text: código fuente (py), HTML, sh, bat...
      ## csv
      Comma-separated-values (casi siempre...)
      : https://docs.python.org/3/library/csv.html
      "Dialectos"
      : https://docs.python.org/3/library/csv.html#dialects-and-formatting-parameters
      ## tsv
      Nombre no muy utilizado (pero por exactitud, se debería utilizar más).
      Se suelen "confundir" con csv

      ## ini
      : https://docs.python.org/3/library/configparser.html
      No hay un estándar definido!
      : https://docs.python.org/3/library/configparser.html#supported-ini-file-structure
      .
      Ejemplo de uso:
        >>> config = configparser.ConfigParser()
        >>> config.read('example.ini')
        ['example.ini']
        >>> config.sections()
        ['language', 'xyz']
        >>> 'language' in config
        True
        >>> config['language']['User']
        'es'

      ## binarios
      "Cualquier" secuencia de bytes
      Lo que "nosotros queramos" que contenga (siguiendo una interpretación det.)
      .
      "Binary itself is meaningless, until such time as an executed algorithm defines
      what should be done with each bit, byte, word or block" -- Wikipedia
      .
      Puede contener trozos que a su vez sean texto (ej. archivos Word)
      Otros ejemplos: audio, imágenes, archivos comprimidos (zip), xls, LibreOffice...
      .
      Herramienta útil: visor / editor hexadecimal

      # Operaciones básicas: open, read, readline,
      # readlines, write, close, with, etc.
      Modos de apertura de un fichero: r, w, a (y b)
      Concepto de "cursor" (posición actual). Acceso secuencial
      Saltos de línea
      "Manejadores" (handles) de fichero
      La importancia de close y el uso de with (...)
      Vídeo (ejemplos sencillos):
      : https://realpython.com/lessons/with-open-pattern/

      Ejemplos con ficheros de texto:
        with open('foo.txt', 'r') as file_read:
            for line in file_read:  # NOTA: mejor que file_read.readlines():
                print(line, end='')  # cada cadena "line" ya termina con "\n"
        with open('foo.txt', 'r') as file_read:
            # De una vez leer todo (sólo si fichero "pequeño"!)
            lista = file_read.readlines()
            # Alternativa: de una vez leer todo (fichero "pequeño"!)
            cad = file_read.read()  # cadena con todo "junto", no lista
        with open('foo.txt', 'r') as file_read:
            while ...:  # podemos hacerlo a mano también, con (o sin) bucles
                line = file_read.readline()
      .
        with open('foo.txt', 'a') as f_write:
            f_write.write('hola\n')
            f_write.writelines(['algo\n', 'mas', 'adios\n'])

      Artículo / tutorial con ejemplos, recomendable:
      : https://realpython.com/read-write-files-python/
      .
      Más avanzado (operaciones con ficheros, copia, directorios...):
      : https://realpython.com/working-with-files-in-python/
      (veremos un poco de esto en el Módulo 10)

      ## time
      Usado para tomar "referencias" de tiempo (timestamps) y medir
      lapsos (intervalos) de tiempo transcurrido (medidos en segundos)
      Ejemplo:
        import time
        time.sleep(5)
        start = time.time()  # guardar fecha/hora actual (segundos)
        ...
        # Calcular y mostrar el lapso de tiempo (ahora - antes)
        print(f"total running time:", time.time() - start, "seconds")

      ## datetime
      Más completo, para gestionar fechas/horas de forma más flexible
      (permite acceder a info de cada "campo" de la fecha: año, mes...) y
      para hacer aritmética. Concepto de timedelta: días + segundos (...)
      Ejemplos:
        import datetime
        now = datetime.date.today()
        print(now.strftime("%m-%d-%y. %d %b %Y is a %A of %B."))
        # dates support calendar arithmetic
        birthday = datetime.date(2001, 1, 1)
        age = now - birthday
        print(age.days, "days old")

      La gestión del tiempo y de fechas/horas no es nada sencilla.
      Los cambios de hora estacionales (DST), zonas horarias, y los formatos
      de representación de fechas, deben gestionarse "bien" (explícitamente!).
      Ejemplo:
        import datetime
        import pytz  # módulo para gestionar las timezones (zonas horarias)
        d_naive = datetime.datetime.now()  # "ingenua": no conoce su timezone
        timezone = pytz.timezone("Europe/Madrid")
        d_aware = timezone.localize(d_naive)  # fecha "consciente" de su tz
        print(d_naive.isoformat())  # isoformat() útil para repr. fechas en logs
        print(d_aware.isoformat())
      .
      Explicación sencilla de lo fundamental sobre timezones:
      https://howchoo.com/g/ywi5m2vkodk/working-with-datetime-objects...

      En ocasiones "bien" significará no hacer "nada": por ejemplo, si
      sabemos que los datos de fechas provienen y se usan de una misma
      zona horaria siempre, y son periodos "cortos" (no verano+inv.) (...)
      Como con otros temas (UTF-8) aunque a veces esa información
      no es explícita, siempre está definida de alguna forma (S.O.?)...
      Lo importante es que la tengamos en cuenta, y no asumamos que
      siempre estamos en el caso "favorable" (sin confirmarlo antes).
      Si hace falta, deberemos añadir parámetros/config. adecuados!
      O si es posible y aplicable, usar "siempre" horas UTC...
        Hora "visible" (usuario)  vs.  "interna" (ej. timestamps) 
      .
      Todo esto es especialmente importante en ciencia de datos (I/O)!

      Más detalles (el módulo datetime cubre más cosas y es más complejo,
      aunque probablemente lo usaremos más, y como tiene solapamientos
      con el módulo time, es mejor empezar / intentar buscar primero si en
      el módulo datetime ya tenemos la funcionalidad que buscamos...)
      .
      Artículo útil y completo, con ejemplos (largo, rec. al menos el inicio):
      : https://realpython.com/python-datetime/
      .
      Artítulo sobre módulo time (avanzado, rec. las dos primeras secciones):

      Documentación oficial de los principales módulos (avanzado, extensa)
      .

      ## exec()
      Avanzado: ejecuta código dinámicamente (en el momento)
      Nota: no confundir con subprocess.run()...
      Ejemplo:
        x = 2
        if x % 2 == 0:
            cad = "formula = x * 5"
        else:
            cad = "formula = 0"
        exec(cad)  # recibe una cadena!
        print(formula)
      .
      Es simplemente una función built-in (no bibl. estándar)
      Similar a eval(), visto en el Módulo 8, pero más general...

      # Concurrencia
      Tema avanzado... sirve para hacer "varias cosas a la vez" (+/-)
      : https://docs.python.org/3/library/concurrency.html
      Asyncio - para I/O; muy avanzado, complejo (...)
      Threading - para I/O o CPU ("turnos": un sólo núcleo!)
      Multiprocessing - para CPU, varios núcleos; más "sencillo"
      aunque se complica para comunicar/compartir info entre ellos
      .
      Más detalles y ejemplos:
      : https://towardsdatascience.com/concurrency-in-python-e770c878ab53
      : https://realpython.com/python-concurrency/
      : https://docs.python.org/3/library/concurrency.html
