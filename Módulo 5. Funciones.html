      # Concepto de función
      Engloban a las funciones matemáticas, y también más cosas
      .
      Conjunto de sentencias que operan sobre un número de entradas
      (parámetros), aunque pueden ser 0, y opcionalmente devuelven
      un resultado (valor, que puede ser compuesto)
      Subprograma o subrutina: "caja negra" con un cierto algoritmo

      # Definir funciones. return
      Ejemplos:
        def myprint(x):
            print(">>", x, end='.')
        
        def multiply(a, b):
            return a * b
        
        def nullfunc():
            pass
      .
      Si no se indica nada, se devuelve None

      # Valor especial None
      Es un valor inmutable, que no es ni True ni False!
        None != True
        None != False
      Se usa para indicar valores o casos "fuera de rango",
      o para valores por defecto

      IMPORTANTE: siempre debe compararse mediante is
        In [30]: id(None)
        Out[30]: 10306432
        
        In [31]: a = None
        
        In [32]: id(a)
        Out[32]: 10306432
        
        In [33]: a == None  # funciona pero no recomendado
        Out[33]: True
        
        In [34]: a is None  # CORRECTO, mejor
        Out[34]: True

      Principio de abstracción (abstracción de [flujo de] control):
      "Each significant piece of functionality in a program should be implemented
      in just one place in the source code. Where similar functions are carried
      out by distinct pieces of code, it is generally beneficial to combine them
      into one by abstracting out the varying parts."
      -- Benjamin C. Pierce in Types and Programming Languages (2002)
      .
      Reutilización: "caja negra" que puede ser modificada/sustituida sin afectarnos
      .
      Pero recordar también el "Principio KISS" (Keep it simple, stupid!)
      : https://en.wikipedia.org/wiki/KISS_principle

      # Parámetros y argumentos
      Parámetros: al definir la función (recibidos)
      Argumentos: al invocarla (enviados / pasados)

      ## Valor por defecto
      Se puede asignar un valor que se tomará
      si se omite el argumento al invocar. Ejemplo:
        def add_vat(base, vat_type=21):
          return base * (100 + vat_type) / 100.0

      ## Posicionales
        def add_vat(base, vat_type=21):
          return base * (100 + vat_type) / 100.0
        add_vat(500, 10)
      ## Por nombre (keyword)
        add_vat(vat_type=10, base=500)
      Es decir, en el segundo caso el que invoca a la función
      debe/puede ser consciente del nombre/signif. del parám.
      .
      IMPORTANTE: siempre primero los argumentos posicionales!

      # Alcance (scope)
      ## global vs. local
      Las variables (entre otras...) pueden ser globales o locales
      Ejemplo:
        x = 20 # global
        def f():
            z = 5  # local
            print(x)
        f()
        print(z)  # ERROR! aquí z no "existe"

      ## shadowing
        x = 20
        def f():
            z = 5
            x = 40
        f()
        print(x)
      Nombres definidos después en otro ámbito pueden "ocultar" o "hacer sombra" a los 1os
      .
      Regla importante para recordar (y simplificar): mejor no reutilizar nombres salvo para
      cosas que sean con certeza temporales (ejemplo: variables para iterar, etc.)
      .
      Más detalles (extenso) en:
      : https://realpython.com/python-namespaces-scope/

      ## Ejemplo: alcance
        # global
        text = 'this is a global variable'
        def justprint():
            print(1, text)  # se puede "leer" la global sin problema
        justprint()
        
        def my_func():
            def enclosing():
                # local
                text = 'this is a local variable'
                print(3, text)
            
            global text  # podremos leer y modificar la global
            print(2, text)
            enclosing()
            text = "modified"
            print(5, text)
        
        my_func()
        print(6, text)

      ## *args y **kwargs

        def f(x, *args, **kwargs):

      # Alcance (cont.)
      ## Composición
      Llamar a una función dentro de otra
        def rectangle_area(base, height):
            return base * height
        def triangle_area(base, height):
            return rectangle_area(base, height) / 2
      .
      O también esto ("encadenar" llamadas a funciones):
        str(int(5.5))

      ## Anidación
      Definir una función dentro de otra!
      Es algo poco habitual pero se puede hacer, sobre todo
      si la función interior es solamente "auxiliar"...
        def triangle_area(base, height):
            def rectangle_area(base, height):
                return base * height
      .
            return rectangle_area(base, height) / 2
      .
      En este caso, rectangle_area() no es visible "fuera"
      Es decir, su alcance/uso está limitado a la 1ª función

      # Recursión o recursividad
      Una función que se llama a sí misma. Ejemplo:
      .
        def factorial(n):
        if n == 0:
            # Define our base (trivial) case
            return 1
        else:
            return n * factorial(n - 1)
        
        print(factorial(5))
      .
      Es otra técnica más de tipo "divide y vencerás" (...)

      Una forma distinta de verlo, que puede ayudar a entenderlo:
      .
        def factorial_0():
            return 1  # caso base o trivial, termina siempre
        
        def factorial_1():
            return 1 * factorial_0()
        
        def factorial_2():
            return 2 * factorial_1()
        
        def factorial_3():
            return 3 * factorial_2()
        
        (...)

      Proceso para diseñar un algoritmo recursivo:
      - Suponer que ya tenemos un algoritmo que nos resuelve una
 "versión reducida" del problema
      - Identificar uno o varios subproblemas "triviales" (casos base)
      - Descomponer el problema en uno o más subproblemas que se
 puedan resolver con el algoritmo que sabemos que ya existe
 y cuya solución vaya aproximándose a los casos base

Crear una función recursiva que calcule la n-ésima potencia

      @CLASS@ ejercicio-6

      La recursión normalmente no hace falta y no se suele usar,
      pero es importante entender cómo funciona.
      .
      IMPORTANTE: es necesario asegurar que la ejecución terminará
      en un número pequeño de niveles de llamada (Python: < 1000).
      Si una función se llama a sí misma indirectamente, es lo mismo
      .
      TODAS las funciones recursivas pueden transformarse en otra
      equivalente con bucles; generalmente más rápida que la que
      era recursiva, pero a veces más compleja de escribir/leer!

      Cuándo es realmente útil / adecuado utilizar la recursión?
      - Cuando el código fuente queda más sencillo y corto
 (pensar en depuración, facilidad de probar y leer...)
      - (Y) Cuando el número de niveles de prof. es "pequeño"
      - (Y) (Opcional, pero deseable) Cuando la solución al
 problema de nivel n-1 es comparativamente bastante
 más rápida (menos costosa) que la del n

      Ejemplos:
      Para recorrer por completo el sistema de ficheros (carpetas dentro de
      carpetas!) o cualquier estructura de datos similar (árboles, etc.)
      Ejemplo para procesar listas de listas recursivamente: aquí (...)
      .
      Para resolver un problema en el que la solcuón en forma recursiva
      sea la más clara (fácil de entender) sin perder mucha eficiencia,
      como por ejemplo ordenar una lista que esté casi ordenada al inicio
        [5, 2, 1, 3, 4,    11, 19, 18, 12, 16]
      .
      Factorial de un número? NO ES EL MEJOR USO, salvo para aprender,
      porque lo de "código sencillo" es opinable, lo del "numero de niveles
      pequeño" no lo cumple (para números grandes), y lo de la rapidez del
      subproblema en realidad tampoco. Lo que indica que es mejor un bucle!

      Otro ejemplo: función que reciba un entero n y nos
      devuelva el n-ésimo número de la serie de Fibonacci.
      : https://es.wikipedia.org/wiki/Serie_de_Fibonacci
      Nota: la serie de Fibonacci es: 0, 1, 1, 2, 3, 5, 8, 13...
      .
        def fibonacci(n):
            if n <= 0:  # Caso base (trivial) 1
                return 0
            elif n == 1:  # Caso base (trivial) 2
                return 1
            else:
                return fibonacci(n-2) + fibonacci(n-1)
      .
      Éste es un ejemplo de uso de recursión por sencillez y brevedad,
      aunque el programa resultante es mucho más lento que un bucle!

      # Iteradores
      Tipo de objetos que representan un stream (flujo) de datos,
      y nos van proporcionando sucesivos elementos
        it = iter([1, 2, 3])  # produce un iterador para la lista
      El iterador es similar a un rango, no podemos verlo "tal cual"
      Pero sí utilizarlos, por ejemplo (y sobre todo) en bucles for
        for i in it:
            ...
      En realidad los estábamos usando siempre, aún sin saberlo:
        for i in [1, 2, 3]:  # de hecho equivale a lo siguiente
        for i in iter([1, 2, 3]):
            ...

      Dónde más se usan: ej. la función list()
      se puede usar para convertir cadenas,
      diccionarios (ojo: sólo las claves),
      tuplas, y otros similares, en una lista.
      En realidad trabaja con iterables
      (iteradores, secuencias, generadores...)
      .
      Va consumiendo el "contenido" del iterador
        list(range(5))
      [0, 1, 2, 3, 4]
        list("hola")

      Lo mismo para otros tipos de colecciones:
        str()  # espera, como las siguientes, un iterable
        tuple()
        set()
        dict()  # Ojo, espera iterables de tuplas clave - valor

      ## Avanzado: enumerate()
      Para comodidad en los bucles, podemos esusar esta función
      que gestiona la variable "contadora" por nosotros
      Ejemplo:
        for i, word in enumerate(['hola', 'que', 'tal']):
            print(f"At index {i} the item is {word}")

      ## Avanzado: módulo itertools
      Tema algo complejo (...) Es un módulo que define
      "functions that “operate” on iterators to
      produce more complex iterators"
      .
      Ejemplo:
        list(zip([1, 2, 3], ['a', 'b', 'c']))
      [(1, 'a'), (2, 'b'), (3, 'c')]
      .
      Nota: zip y map (lo veremos) son "built-ins"
      En itertools tenemos otros similares para otras cosas

      Otro ejemplo:
      Iterador que nos da los números impares: se
      comporta similar a range() pero sin fin...
        import itertools
        odds = itertools.count(start=1, step=2)
        while True:
            print(next(odds))
