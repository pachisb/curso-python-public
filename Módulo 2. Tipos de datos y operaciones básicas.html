      ## Operadores relacionales
        == !=  # comparación de igualdad / desigualdad
        < > >= <=
      Comparan el *valor* de dos objetos, que no tienen
      por qué ser del mismo tipo (hay conversiones...)
      Devuelven un booleano: True o False
      .
      Nota: no existen los operadores <> (Python 2), => ni =<

      Diferencias con otros lenguajes:
      - Todos tienen la misma prioridad, que es inferior a
 la de los operadores aritméticos
      - Se permiten expresiones como a < b < c con el
 significado habitual en matemáticas. Es decir, se
 pueden encadenar n comparaciones (se evalúan
 por pares, de izda. a dcha.)
      .
      Más detalles:
      : https://docs.python.org/3/reference/expressions.html#comparisons

      ## Evaluación cortocircuitada
      Se basa en simplificar expresiones:
        x > 5 or x <= 5 or x > 2
      True
      No hace falta llegar al tercer término (x > 2) si alguno
      anterior ya resultó ser True (y estamos en un or)
      .
        x < 5 and False and 5 > 0
      False

      Otra forma de verlo:
      Una expresión booleana deja de evaluarse cuando el
      resultado es "definitivo" (ya no hay cambio posible):
      - El and se "detiene" al encontrar un False
      - El or se "detiene" al encontrar un True

      ## Evaluación cortocircuitada (cont.)
      Posibles problemas "ocultos" en nuestro código fuente:

      # Precedencia de operadores
      Aritméticos: regla "*PEMDAS*"
        P   E  MD  AS
        ( ) ** * / + -
      Notas:
      - M y D igual prioridad entre sí; A y S también.
      - Ojo que los signos (+ y -) no son AS, aunque a veces lo parezca
      - Multiplicación de matrices (@) y otras divisiones (// %) son como MD
      - M, D, A y S se evalúan de izda. a dcha. (es lo habitual)
      - Exponentes se evalúan de dcha. a izda.! (Ej. 4 3 2)
      : http://www.mathsisfun.com/operation-order-pemdas.html

      Grupos principales (dentro de algunos de ellos, el orden ya descrito antes):
      - Primero los paréntesis (P) y similares (corchetes, llaves...)
      - Luego el exponente (E)
      - Luego los signos (+ y -); y luego la negación de bits (~)
      - Luego los MD; seguidos de los AS
      - Luego el resto de operaciones de manipulación de bits (binarias)
      - Luego los relacionales (comparaciones), incluyendo variantes: is, in...
      - Luego los operadores lógicos (not, and y or; en ese orden)
      - Luego el operador ternario
      Tabla completa y más detalles:
      : https://docs.python.org/3/reference/expressions.html#operator-precedence

      # Variables y constantes
      Variable = dato con un valor asociado que puede cambiar,
      siempre dentro de un determinado tipo de dato, con un
      identificador (nombre) y que tiene un ámbito (alcance)
        x = 5
        x = "hola"
      En Python el tipo de dato puede cambiarse y no hace falta
      declarar ("definir" la variable de antemano) porque es un
      lenguaje "no fuertemente tipado"
      En otros lenguajes, una variable "es" una posic. en memoria

      Constante = es un dato cuyo valor es inalterable
        PI = 3.1415927
      .
      Un literal también lo es, pero normalmente llamamos
      constante a los datos con un identificador asociado
        3.1415927
      .
      Ventajas de usar constantes: legibilidad, mantenibilidad,
      y permiten al lenguaje optimizar cosas

      Identificadores válidos
      (...) "the valid characters for identifiers are the same as
      in Python 2.x: the uppercase and lowercase letters A through
      Z, the underscore _ and, except for the first character, the
      digits 0 through 9"

      Algunas recomendaciones:
      - Es habitual usar sólo minusculas, underscore y dígitos
      - Es habitual reservar las mayúsculas para CONSTANTES
      - Que sean descriptivos aunque mejor si son breves
      - Evitar 'l', 'I', 'O', y palabras como list, str...
      - No usar acentos, eñes, ç, etc.

      Ejemplos:
        a = 5
        PI = 3.1415927
      Nota: en realidad se permiten otros caracteres también. Ej.:
        año = 2020
      .
      
      Ver más detalles en el Módulo 4
      .
      Avanzado: reglas para los identificadores
      : https://docs.python.org/3/reference/lexical_analysis.html#identifiers

      Asignación: asociar un [nuevo] valor (de un determinado
      tipo) a una variable. Ej.:
        nombre_variable = <expresión>
      .
      Primero se evalúa la expresión, y el valor resultante se
      "guarda" (en realidad se asocia, en Python) en la variable

      En Pyhton, la asignación usa el símbolo de igualdad (=)
      pero no se debe confundir con la notación y el significado
      que se le dan en matemáticas. Ejemplo:
        x = x + 1
      No confundir tampoco con el operador de igualdad (==), que
      en otros lenguajes usa el mismo símbolo (=), más confuso...
      .
      La asignación es por tanto una sentencia, no una expresión
      Nota: NO es ambas cosas tampoco (no devuelve un valor)
      a diferencia de lo que pasa en algunos otros lenguajes!

      Asignaciones múltiples:
        var2 = var1 = 5
      Importante: el orden (de asignación) es de dcha. a izda.
      .
        x, y = y, x
      Ésta última de hecho es la forma recomendada para
      "intercambiar" el valor de 2 o n elementos
      Importante: cómo se haría con asignaciones simples?

      Asignaciones especiales (con operaciones):
        x = 1
        x *= 15  # equivale a x = x * 15
        x += 10  # equivale a x = x + 10
        x //= 5  # división entera
        x /= 5  # ojo, división flotante (cambia el tipo)
        x -= 3
        x **= 2  # cuadrado!
        etc.

      ## str.format()
      En otros lenguajes es habitual algo como:
        printf("El resultado es: %d", 5)
      Nota: la notación tipo printf la veremos (operador %)
      pero ya no es habitual/recomendable usarla en Python 3
      .
      Y también hay un format() built-in similar (pero menos claro):
        print("El resultado es: " + format(5, "d"))

      Es más cómodo/potente/flexible utilizar str.format(),
      que además permite convertir tipos y más cosas (...):
        "x = {}, y = {}, z = {}".format(x, y, z)
        "{:f} = {} / {}".format(10, 200, 20)
        "{first:f} {second} {first}".format(first=10, second=20)
      .
      Pensar en la cadena como una "plantilla" para el resultado final
      Sintaxis, formatos disponibles, etc.:
      : https://docs.python.org/3/library/string.html#formatspec

      ## f-strings (Python >= 3.6)
      Aunque desde que existen los "formatted string literals"
      (f-strings), son lo más utilizado (y con motivo!)
        a = 10
        b = 20
        c = 30
        f"{c:f}, {b}, {a}"
      .
      Las variables y expresiones se reemplazan automáticamente!
      Por lo demás, todo igual que con str.format()
      El resultado es una cadena normal. Más detalles:
      : https://realpython.com/python-f-strings/
